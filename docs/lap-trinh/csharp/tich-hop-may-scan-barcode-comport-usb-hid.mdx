Khi tích hợp máy scan barcode vào phần mềm C# Windows Forms, có hai cách kết nối phổ biến nhất:

* **USB HID (Human Interface Device)**: máy scan giả lập bàn phím, gửi ký tự trực tiếp vào ứng dụng.
* **COM Port (RS-232 / RS-485)**: máy scan giao tiếp serial, ứng dụng nhận chuỗi dữ liệu qua cổng COM.

Mỗi cách có ưu nhược điểm riêng và phù hợp với các tình huống khác nhau. Bài viết này hướng dẫn triển khai cả hai, từ cách nhận data đến các kỹ thuật xử lý thực tế.

### So sánh COM Port vs USB HID

| Tiêu chí | COM Port (RS-232/RS-485) | USB HID |
| :--- | :--- | :--- |
| **Nguyên lý** | Giao tiếp serial – nhận chuỗi ký tự qua cổng COM | Giả lập bàn phím – gửi keystroke trực tiếp vào hệ thống |
| **Nhận data trong C#** | `SerialPort.DataReceived` event | TextBox `KeyDown` / `KeyPress` event |
| **Cần driver riêng?** | Có thể cần (CP2102, CH340...) | Không – Windows tự nhận HID |
| **Cấu hình BaudRate, Parity...** | Bắt buộc phải khớp với thiết bị | Không cần |
| **Data đến từng mảnh?** | Có – cần buffer ghép dòng | Không – mỗi keystroke là 1 ký tự |
| **Thread safety** | Phải Invoke về UI thread | Tự động trên UI thread |
| **Phù hợp khi** | Cần kiểm soát tốc độ, giao thức, môi trường công nghiệp | Đơn giản, cắm vào dùng ngay, ít cấu hình |

### USB HID Scanner – Cách hoạt động & tích hợp

Máy scan USB HID hoạt động như một bàn phím ảo. Khi scan xong, nó tự động gửi chuỗi ký tự + ký tự kết thúc (thường là Enter) vào control đang được focus. Không cần viết code nhận data – Windows xử lý tầng HID tự động.

#### Cách 1 – Bắt KeyDown trên TextBox

Cách đơn giản nhất: đặt focus vào TextBox, máy scan sẽ gõ thẳng vào đó. Bắt sự kiện `KeyDown` để xử lý khi nhấn Enter:

```csharp
// Bắt KeyDown trên TextBox – cách đơn giản nhất
private void txtBarcode_KeyDown(object sender, KeyEventArgs e)
{
    if (e.KeyCode != Keys.Enter) return;

    e.Handled = true;
    e.SuppressKeyPress = true;  // Chặn tiếng 'ding' khi nhấn Enter

    string barcode = txtBarcode.Text.Trim();
    if (string.IsNullOrEmpty(barcode)) return;

    ProcessBarcode(barcode);
    txtBarcode.Clear();
}
```

#### Cách 2 – Bắt toàn cục qua Form.KeyPreview

Khi muốn nhận barcode toàn cục (không cần focus TextBox cụ thể), dùng `KeyPreview` ở Form. Dùng kỹ thuật **timing** để phân biệt input từ máy scan (rất nhanh) vs người dùng gõ tay (chậm hơn):

```csharp
public MyForm()
{
    InitializeComponent();
    this.KeyPreview = true;  // Form nhận key trước controls
    this.KeyPress += Form_KeyPress;
}

private StringBuilder _inputBuffer = new StringBuilder();
private DateTime _lastKeyTime = DateTime.MinValue;

private void Form_KeyPress(object sender, KeyPressEventArgs e)
{
    // Máy scan gõ rất nhanh (< 50ms/ký tự)
    // Người dùng gõ chậm hơn → dùng timing để phân biệt
    if ((DateTime.Now - _lastKeyTime).TotalMilliseconds > 100)
        _inputBuffer.Clear();

    _lastKeyTime = DateTime.Now;

    if (e.KeyChar == '\r')  // Enter = kết thúc barcode
    {
        string barcode = _inputBuffer.ToString().Trim();
        if (barcode.Length > 0) ProcessBarcode(barcode);
        _inputBuffer.Clear();
    }
    else
    {
        _inputBuffer.Append(e.KeyChar);
    }
}
```

⚠️ **Lưu ý timing:** Ngưỡng 100ms phù hợp cho hầu hết máy scan. Nếu máy scan của bạn có tốc độ chậm hơn, tăng lên 150–200ms. Ngưỡng quá thấp sẽ bị nhầm với người gõ tay nhanh.

#### ✅ Ưu điểm
* Cắm vào dùng ngay, không cần cấu hình.
* Windows tự nhận driver, không cài thêm.
* Code đơn giản, data đến đầy đủ.
* Không lo thread safety.

#### ❌ Nhược điểm
* Phải đảm bảo focus đúng control.
* Không kiểm soát được tốc độ / giao thức.
* Cần cấu hình suffix (Enter) từ máy scan.

### COM Port Scanner – Cách hoạt động & tích hợp

Máy scan COM Port giao tiếp theo giao thức serial (RS-232). Dữ liệu được truyền theo các thông số: BaudRate, Parity, DataBits, StopBits – phải cấu hình khớp giữa máy scan và phần mềm. Dữ liệu đến **bất đồng bộ trên một thread riêng**, không phải UI thread.

#### Hàm SetSerialPortLookUp – Lấy danh sách cổng COM

Trước khi kết nối, lấy danh sách cổng COM có trên máy để cho user chọn. Dùng `SerialPort.GetPortNames()`:

```csharp
private void SetSerialPortLookUp()
{
    DataTable dtPort = new DataTable();
    dtPort.Columns.Add("Port", typeof(string));

    // Lấy toàn bộ cổng COM có trên máy (VD: COM1, COM3, COM7...)
    foreach (string portName in SerialPort.GetPortNames())
        dtPort.Rows.Add(portName);

    // Bind vào ComboBox để user chọn
    cboComport.Properties.DataSource    = dtPort;
    cboComport.Properties.ValueMember   = "Port";
    cboComport.Properties.DisplayMember = "Port";

    // Mặc định chọn cổng đầu tiên
    if (dtPort.Rows.Count > 0)
        cboComport.EditValue = dtPort.Rows[0][0].ToString();
}
```

**Lưu ý:** `GetPortNames()` trả về mảng các cổng đang có trên hệ thống. Danh sách này thay đổi khi cắm/rút thiết bị USB-to-Serial – nên gọi lại hàm này nếu user cắm thêm thiết bị sau khi form đã load.

#### Hàm ConnectSerialPort – Khởi tạo & mở cổng COM

Bước quan trọng nhất: cấu hình các thông số serial phải **khớp chính xác** với máy scan. Sai 1 thông số là không nhận được data:

```csharp
private SerialPort _serialPort = null;

private void ConnectSerialPort()
{
    try
    {
        DisconnectSerialPort(); // Đóng port cũ trước khi mở mới

        string selectedPort = cboComport.EditValue?.ToString();
        if (string.IsNullOrEmpty(selectedPort)) return;

        _serialPort = new SerialPort()
        {
            PortName     = selectedPort,
            BaudRate     = 9600,              // Tốc độ truyền – phải khớp máy scan
            Parity       = Parity.Even,       // Kiểm tra chẵn lẻ
            DataBits     = 7,                 // Số bit dữ liệu
            StopBits     = StopBits.Two,      // Số stop bit
            Handshake    = Handshake.None,
            Encoding     = Encoding.UTF8,
            NewLine      = "\r",              // Ký tự kết thúc dòng
            WriteTimeout = 1000
        };

        _serialPort.DataReceived += serialPort_DataReceived;
        _serialPort.Open();

        lblStatus.Text      = "Connected " + selectedPort;
        lblStatus.BackColor = Color.Green;
    }
    catch (Exception ex)
    {
        lblStatus.Text      = "Disconnect";
        lblStatus.BackColor = Color.Gray;
        Debug.WriteLine("Serial port connect failed: " + ex.Message);
    }
}

private void DisconnectSerialPort()
{
    try
    {
        if (_serialPort != null)
        {
            if (_serialPort.IsOpen) _serialPort.Close();
            _serialPort.DataReceived -= serialPort_DataReceived;
            _serialPort.Dispose();
            _serialPort = null;
        }
    }
    catch (Exception ex) { Debug.WriteLine(ex.Message); }

    lblStatus.Text      = "Disconnect";
    lblStatus.BackColor = Color.Gray;
}
```

⚠️ **Thông số serial phải tra manual máy scan.** Sai 1 thông số sẽ nhận được ký tự rác hoặc không nhận được gì. 3 thông số phổ biến nhất: **9600/N/8/1** → **9600/E/7/2** → **115200/N/8/1**. Thử lần lượt nếu không biết trước.

#### Hàm serialPort_DataReceived – Nhận & ghép buffer

Đây là phần kỹ thuật phức tạp nhất. Khác với USB HID, data serial **có thể đến từng mảnh nhỏ** (không đủ 1 barcode trong 1 lần nhận). Cần dùng `StringBuilder` làm buffer để ghép lại:

```csharp
// Buffer ghép dữ liệu serial đến từng mảnh
private readonly StringBuilder _serialBuffer = new StringBuilder();
private readonly object _bufferLock = new object();

private void serialPort_DataReceived(object sender, SerialDataReceivedEventArgs e)
{
    try
    {
        string received = _serialPort.ReadExisting();
        if (string.IsNullOrEmpty(received)) return;

        lock (_bufferLock)  // Thread-safe: event chạy trên thread riêng
        {
            _serialBuffer.Append(received);

            // Tách từng dòng hoàn chỉnh (kết thúc bằng CR hoặc LF)
            while (true)
            {
                string bufStr = _serialBuffer.ToString();
                int idx = bufStr.IndexOfAny(new char[] { '\r', '\n' });
                if (idx < 0) break;  // Chưa có dòng hoàn chỉnh → đợi

                string line = bufStr.Substring(0, idx).Trim();

                // Xử lý \r\n hoặc \n\r bị tách đôi
                int skip = 1;
                if (idx + 1 < bufStr.Length &&
                    (bufStr[idx + 1] == '\r' || bufStr[idx + 1] == '\n') &&
                    bufStr[idx + 1] != bufStr[idx])
                    skip = 2;

                _serialBuffer.Remove(0, idx + skip);

                if (!string.IsNullOrEmpty(line))
                {
                    string captured = line; // Tránh closure bug
                    if (this.IsHandleCreated && !this.IsDisposed)
                        this.BeginInvoke(new Action(
                            () => ProcessScannedCode(captured)));
                }
            }
        }
    }
    catch (Exception ex)
    {
        Debug.WriteLine("Serial read error: " + ex.Message);
    }
}
```

**Giải thích logic buffer ghép dòng:**
* `ReadExisting()` đọc tất cả data hiện có – có thể chỉ là *một phần* của barcode.
* Append vào `StringBuilder` → kiểm tra `IndexOfAny(CR/LF)` → nếu chưa có thì đợi data tiếp theo.
* Khi có CR/LF → cắt ra 1 dòng hoàn chỉnh → xử lý → tiếp tục kiểm tra phần còn lại trong buffer.
* `lock(_bufferLock)` đảm bảo thread-safe vì `DataReceived` chạy trên thread riêng của SerialPort.

#### Hàm ProcessScannedCode – Phân loại & xử lý mã scan

Sau khi nhận được chuỗi hoàn chỉnh, phân loại và xử lý theo nghiệp vụ. Hàm này đã được `BeginInvoke` về UI thread nên an toàn cập nhật controls trực tiếp:

```csharp
private void ProcessScannedCode(string scannedCode)
{
    // Đã Invoke về UI thread → an toàn cập nhật controls

    if (scannedCode.StartsWith("DOC", StringComparison.OrdinalIgnoreCase))
    {
        // Mã chứng từ → load danh sách liên quan
        ProcessDocumentScan(scannedCode);
    }
    else
    {
        // Mã barcode sản phẩm → verify / tìm kiếm
        txtBarcode.Text = scannedCode;
        VerifyBarcode(scannedCode);
    }
}
```

#### Quản lý kết nối theo vòng đời Form

Chỉ nên giữ cổng COM khi form đang active, tránh chiếm cổng khi chuyển sang form khác hoặc mở ứng dụng khác:

```csharp
// Kết nối khi form được focus
private void MyForm_Activated(object sender, EventArgs e)
    => ConnectSerialPort();

// Ngắt khi form mất focus
private void MyForm_Deactivate(object sender, EventArgs e)
    => DisconnectSerialPort();

// Dọn dẹp khi đóng form
protected override void OnFormClosing(FormClosingEventArgs e)
{
    DisconnectSerialPort();
    base.OnFormClosing(e);
}
```

⚠️ **Tại sao cần Activated/Deactivate?** Nếu giữ cổng COM mở suốt, các ứng dụng khác (hoặc form khác trong cùng ứng dụng) sẽ không thể mở cổng đó và nhận lỗi *"Access to the port is denied"*. Cách xử lý này giải quyết triệt để vấn đề.

### Lưu ý kỹ thuật nâng cao

#### 1. Data serial đến từng mảnh – Đừng bỏ qua buffer
Lỗi phổ biến của người mới dùng COM Port là giả định rằng một lần kích hoạt sự kiện `DataReceived` sẽ trả về một barcode hoàn chỉnh. Thực tế không phải vậy. Tùy thuộc vào BaudRate, độ dài dữ liệu và cơ chế lập lịch của hệ điều hành, dữ liệu có thể bị xé lẻ thành 2-3 đợt truyền. Việc không sử dụng buffer sẽ dẫn đến tình trạng barcode bị đứt gãy và xử lý sai thông tin. Luôn sử dụng giải pháp ghép dòng với `StringBuilder`.

#### 2. Thread Safety – Bắt buộc Invoke về UI thread
Sự kiện `DataReceived` chạy trên một luồng phụ do hệ thống quản lý. Mọi nỗ lực cập nhật trực tiếp giao diện (như ghi đè text vào TextBox) từ luồng này sẽ gây lỗi hệ thống `Cross-thread operation not valid`. Sử dụng phương thức `BeginInvoke` (bất đồng bộ) thay vì `Invoke` (đồng bộ) để loại bỏ nguy cơ deadlock.

#### 3. Xử lý CR/LF hỗn hợp
Tùy vào cấu hình của từng loại máy quét, ký tự kết thúc có thể được thiết lập là `CR (\r)`, `LF (\n)` hoặc cặp `CRLF (\r\n)`. Cần lập trình bộ lọc linh hoạt để tương thích với tất cả định dạng trên mà không làm nhân đôi dữ liệu.

```csharp
// Phát hiện CR hoặc LF
int idx = bufStr.IndexOfAny(new char[] { '\r', '\n' });

// Bỏ qua ký tự thứ 2 của cặp CRLF / LFCR
int skip = 1;
if (idx + 1 < bufStr.Length &&
    (bufStr[idx + 1] == '\r' || bufStr[idx + 1] == '\n') &&
    bufStr[idx + 1] != bufStr[idx])  // Khác ký tự → là cặp CRLF
    skip = 2;
```

#### 4. Cách kiểm tra lỗi khi lệch thông số Serial
Nếu dữ liệu trả về bị mã hóa thành các ký tự lạ hoặc hoàn toàn không phản hồi:
* Sử dụng các phần mềm chuyên dụng như **PuTTY** hoặc **Termite** để kiểm tra trực tiếp cổng COM. Nếu các công cụ này nhận diện đúng mà ứng dụng của bạn bị lỗi, nguyên nhân nằm ở mã nguồn C#. Nếu cả hai đều hiển thị ký tự rác, cấu hình phần cứng đang bị lệch.
* Tra cứu sách hướng dẫn đi kèm của máy quét để lấy thông số mặc định.
* Thử nghiệm tuần tự các cấu hình phổ biến: `9600/N/8/1` → `9600/E/7/2` → `115200/N/8/1`.

#### 5. Lỗi Closure Bug trong BeginInvoke
Hãy cẩn thận khi sử dụng biểu thức Lambda bên trong các vòng lặp xử lý dữ liệu. Nếu không tách biến, giá trị truyền vào hàm xử lý có thể bị thay đổi trước khi tiến trình cập nhật giao diện kịp thực thi:

```csharp
// ❌ SAI: line bị capture by reference
// Đến khi BeginInvoke thực thi, line đã là giá trị mới
this.BeginInvoke(new Action(() => Process(line)));

// ✅ ĐÚNG: copy ra biến local trước khi capture
string captured = line;  // Snapshot giá trị tại thời điểm này
this.BeginInvoke(new Action(() => Process(captured)));
```

### Kết luận – Khi nào chọn COM, khi nào chọn USB?

#### 🔌 Chọn USB HID khi...
* Ứng dụng đơn giản, chỉ cần quét và lấy dữ liệu text nhanh chóng.
* Cần triển khai nhanh, cắm thiết bị vào máy là có thể vận hành ngay.
* Môi trường văn phòng, cửa hàng bán lẻ ít chịu nhiễu tín hiệu.

#### 🔧 Chọn COM Port khi...
* Cần kiểm soát luồng dữ liệu, phân loại nghiệp vụ phức tạp trực tiếp từ phần cứng.
* Vận hành trong môi trường nhà máy, xí nghiệp sử dụng tiêu chuẩn RS-232/RS-485.
* Cần kết nối đồng thời nhiều thiết bị ngoại vi vào hệ thống.

Việc tách biệt rõ ràng giữa lớp nhận dữ liệu (Data Receiver Layer) và lớp xử lý nghiệp vụ (`ProcessScannedCode`) sẽ giúp hệ thống của bạn linh hoạt, dễ dàng chuyển đổi phương thức kết nối phần cứng mà không ảnh hưởng tới logic vận hành cốt lõi.
